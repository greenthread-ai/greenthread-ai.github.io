Custom guardrails your way

Build guardrails that match your exact requirements—run them locally or call external services.

When prepackaged guardrails aren't enough, create custom ones. Implement guardrail logic directly within GreenThread, or integrate with external systems via remote API calls. Your proprietary validation logic, your industry-specific rules, your compliance requirements—all seamlessly integrated into the inference pipeline.

Internal Implementation

Write custom logic that runs within GreenThread for minimal latency

Remote System Calls

Call external APIs for specialized validation or third-party services

custom-guardrail.yaml 
guardrail:
  name: credit-card-detector
  type: input
  action: block_and_alert

  rules:
    - pattern: credit_card
      sensitivity: high
      on_match:
        - redact
        - log_event
        - notify: compliance@company.com

  remote_validation:
    endpoint: https://compliance.internal/validate
    timeout: 100ms
    on_failure: allow_with_flag

Granular policy controls

Define what's allowed and what's blocked at every level—from network access to tool usage.

Policies give you fine-grained control over model behavior and system access. Restrict MCP tool usage to approved environments only. Limit inbound requests to specific IP ranges. Control which external resources models can access. Policies are enforced at the platform level, providing a security layer that can't be bypassed by prompt manipulation.

  • Restrict MCP tooling to specific environments
  • IP allowlisting for inbound request filtering
  • Outbound connection restrictions per model instance
  • Role-based access controls for model management
  • Time-based policies for scheduled restrictions

Payment Card Protection

A customer service chatbot needs to help users but must never process or store credit card numbers.

Input Guardrail PCI pattern detection with auto-redaction
Policy Block requests containing card patterns
Alert Notify compliance team immediately
Result: Card numbers are redacted before reaching the model. User receives helpful guidance to use secure payment portal instead. Compliance team gets instant notification for review.

Environment Isolation

An enterprise needs their AI agent to use MCP tools in production, but wants staging completely isolated from external systems.

Production Policy MCP tools enabled, approved endpoints only
Staging Policy All MCP outbound connections denied
Logging All policy decisions logged for audit
Result: Developers can safely test in staging without risk of accidental external calls. Production maintains full functionality with controlled access.
